# 看板模块买卖建议设计

## 背景

当前看板模块已经提供以下能力：

- 实时行情摘要；
- 趋势结论（上涨趋势 / 下跌趋势 / 横盘震荡）；
- 技术指标分析（MA、MACD、BOLL、RSI、量能、形态、资金）；
- 多张辅助图表。

现阶段分析结果停留在“描述市场状态”，尚未给出更贴近交易决策的展示型建议。用户希望在不引入外部模型和额外数据源的前提下，基于现有指标数据，补充：

- 买入 / 卖出 / 观望建议；
- 目标价位；
- 建议持续时间；
- 建议依据与风险提示。

## 目标

- 在现有分析结果之上新增一层“展示型建议”，输出买入 / 卖出 / 观望结论；
- 所有建议仅基于现有服务端分析已具备的数据与指标推导，不新增外部接口；
- 给出可解释的目标价位、止损 / 失效价、建议持续时间和依据说明；
- 保持现有异步请求、轮询、刷新、错误态与图表渲染逻辑不变；
- 数据不足或指标冲突时，默认回退到可恢复、可解释的观望建议。

## 非目标

- 不实现自动交易或下单能力；
- 不引入机器学习模型、AI 预测接口或新的第三方行情源；
- 不新增复杂仓位管理逻辑，如加仓 / 减仓 / 分批建仓；
- 不重构现有行情请求、轮询、缓存或页面布局；
- 不把建议包装为“确定性结论”，仍然定位为辅助判断信息。

## 方案概览

在 `server/analysis.js` 的既有趋势分析结果中新增 `advice` 字段，由纯函数基于当前指标结果同步推导。前端 `src/components/TrendDashboard.vue` 在现有“趋势结论”区域附近增加一张“操作建议卡”，展示建议动作、置信度、目标价、止损 / 失效价、建议持续时间、依据标签、完整说明和风险提示。

该方案不新增请求、不新增事件监听、不新增定时器，仅扩展服务端返回结构和前端展示层。

## 输出结构

在 `TrendAnalysisPayload["analysis"]` 下新增 `advice`：

```ts
advice: {
  action: "买入" | "卖出" | "观望";
  confidence: "高" | "中" | "低";
  targetPrice: number | null;
  stopPrice: number | null;
  holdingWindow: string;
  reasonTags: string[];
  rationale: string;
  riskNote: string;
}
```

字段约束如下：

- `action`：最终建议动作，仅允许买入 / 卖出 / 观望；
- `confidence`：综合信号一致性后的置信度；
- `targetPrice`：建议目标价；观望时允许为空；
- `stopPrice`：建议止损价或建议失效价；观望时允许为空；
- `holdingWindow`：建议观察或持有周期，如 `3-5个交易日`、`1-2周`；
- `reasonTags`：从关键指标映射出的短标签，供前端高亮；
- `rationale`：完整说明，解释结论如何得出；
- `riskNote`：说明何种场景下当前建议失效或需要重新评估。

## 规则设计

### 总体原则

- 只使用现有指标：MA、MACD、BOLL、RSI、量能、K 线形态、主力资金、近 20 日高低点；
- 建议生成必须为纯函数，不依赖额外异步状态；
- 信号冲突时优先保守，宁可给观望，不强行给方向；
- 横盘震荡优先级高于弱趋势，避免在整理期给出激进结论；
- 所有建议都必须能解释“为什么是这个动作”以及“何时失效”。

### 信号归类

将已有指标按方向性划分为三类：

- 偏多信号：`ma.bias === "偏多"`、`macd.bias === "偏多"`、`rsi.bias === "偏多"`、`volume.relation === "价涨量增"`、`pattern.pattern === "突破"`、`capital.mainForceNetAmount >= 0`；
- 偏空信号：`ma.bias === "偏空"`、`macd.bias === "偏空"`、`rsi.bias === "偏空"`、`volume.relation === "价跌量增"`、`pattern.pattern === "破位"`、`capital.mainForceNetAmount < 0`；
- 中性 / 震荡信号：横盘命中数较高、BOLL 收口、均线粘合、量能萎缩、RSI 位于 40-60。

### 动作判定

建议动作按以下优先级推导：

1. 若 `sidewaysSignals.length >= 3`，直接输出 `观望`；
2. 若偏多核心信号同时成立：
   - `ma.bias === "偏多"`
   - `macd.bias === "偏多"`
   - `rsi.bias === "偏多"`
   则基础动作判为 `买入`；
3. 若偏空核心信号同时成立：
   - `ma.bias === "偏空"`
   - `macd.bias === "偏空"`
   - `rsi.bias === "偏空"`
   则基础动作判为 `卖出`；
4. 若未满足核心共振，则根据辅助信号数量判断：
   - 偏多辅助信号明显占优，输出 `买入`；
   - 偏空辅助信号明显占优，输出 `卖出`；
   - 否则输出 `观望`。

辅助信号数量“明显占优”的建议标准：

- 偏多数量至少比偏空数量多 2 项，判 `买入`；
- 偏空数量至少比偏多数量多 2 项，判 `卖出`；
- 其他情况均判 `观望`。

### 置信度判定

`confidence` 由信号一致性决定：

- 高：核心三项共振成立，且突破 / 破位、量价关系、资金方向中至少有 2 项同向；
- 中：核心三项未完全共振，但方向性信号占优明显；
- 低：横盘、冲突信号较多、或仅依赖少量辅助信号得出结果。

### 目标价推导

目标价全部基于现有技术位推导，不使用固定涨跌百分比拍脑袋生成。

#### 买入目标价

从以下候选位中选择“更近的有效压力位”作为目标价：

- BOLL 上轨；
- 近 20 日最高价；
- 若实时价已接近或突破近 20 日高点，则目标价可取 `实时价` 与 `BOLL 上轨`、`近 20 日高点` 中更高的一档。

买入目标价约束：

- 不低于当前价；
- 若候选位全部低于或过于接近当前价，则按“数据不具备明确空间”降级为观望，避免给出无效买点。

#### 卖出目标价

从以下候选位中选择“更近的有效支撑位”作为目标价：

- BOLL 下轨；
- 近 20 日最低价；
- 若实时价已接近或跌破近 20 日低点，则目标价可取 `实时价` 与 `BOLL 下轨`、`近 20 日低点` 中更低的一档。

卖出目标价约束：

- 不高于当前价；
- 若候选位全部高于或过于接近当前价，则按“下行空间不清晰”降级为观望。

#### 观望目标价

- 不给单点目标价；
- `targetPrice` 设为 `null`；
- 在 `rationale` 中说明当前处于等待突破 / 破位确认阶段；
- 如有必要，可引用近 20 日高低点形成“观察区间”描述，但不新增结构字段。

### 止损 / 失效价推导

- 买入：取 `MA20`、`BOLL 中轨`、近 20 日低点中的较近支撑位，作为止损 / 建议失效价；
- 卖出：取 `MA20`、`BOLL 中轨`、近 20 日高点中的较近压力位，作为止损 / 建议失效价；
- 观望：不设置止损，返回 `null`。

约束：

- 买入止损价必须低于当前价；
- 卖出止损价必须高于当前价；
- 若无法满足上述条件，则当前建议降级为观望，避免出现逻辑相反的价位。

### 持续时间推导

建议持续时间由趋势强弱决定：

- 强共振趋势：`1-2周`
- 一般趋势：`3-5个交易日`
- 横盘 / 观望：`1-3个交易日观察`

判定口径：

- 若核心三项共振且辅助同向信号充足，则视为强共振趋势；
- 若只有部分趋势信号成立，则视为一般趋势；
- 横盘或冲突场景统一进入观察期。

### 依据与风险文案

`reasonTags` 从有效信号中提取，最多展示 4-6 项，优先顺序如下：

- 均线排列状态；
- MACD 状态；
- RSI 状态；
- BOLL 位置或收口 / 开口；
- 量价关系；
- 形态（突破 / 破位 / 横盘）；
- 主力资金方向。

`rationale` 生成原则：

- 买入：说明上涨共振或突破依据，解释目标价对应的技术位；
- 卖出：说明下跌共振或破位依据，解释目标价对应的支撑位；
- 观望：说明横盘或冲突原因，并提示等待下一步确认。

`riskNote` 生成原则：

- 买入：跌破止损 / 失效位、量价背离加剧、MACD 反向死叉时需重新评估；
- 卖出：反抽站回失效位、放量止跌、MACD 金叉时需重新评估；
- 观望：突破或跌破观察区间后重新判断。

## 异常与回退策略

以下情况统一回退为 `观望 + 低置信度`：

- 指标关键值缺失；
- 目标价或止损价无法推导出合理方向；
- 多空信号数量接近，无法形成有效倾向；
- 数据长度不足以支撑关键判断。

回退后的输出要求：

- `action = "观望"`
- `confidence = "低"`
- `targetPrice = null`
- `stopPrice = null`
- `holdingWindow = "1-3个交易日观察"`
- `rationale` 明确说明“信号不足或冲突，暂不建议直接操作”
- `riskNote` 提示关注突破 / 破位后的再判断

## 前端展示方案

### 布局位置

在 `src/components/TrendDashboard.vue` 的趋势结论区域中新增“操作建议卡”，优先放置在趋势结论卡附近，避免用户在多个区域跳读。

### 展示内容

建议卡展示以下信息：

- 建议动作；
- 置信度；
- 目标价位；
- 止损 / 失效价；
- 建议持续时间；
- 依据标签；
- 完整建议说明；
- 风险提示。

### 视觉语义

- `买入` 使用偏多色；
- `卖出` 使用偏空色；
- `观望` 使用中性色；
- 文案风格保持当前看板一致，强调“建议”而非“指令”。

## 类型与实现边界

### 服务端

仅在 `server/analysis.js` 增加建议推导函数并挂入返回结果：

- 新增建议生成纯函数；
- 复用现有分析结果，不重复计算已有指标；
- 不修改现有外部接口调用顺序，不新增并发请求。

### 前端

仅做展示型接入：

- 扩展 `src/market-types.ts` 中的类型定义；
- 在 `src/components/TrendDashboard.vue` 中新增建议卡渲染；
- 复用现有 loading / error / retry 机制。

## 异步与稳定性分析

本次设计不触碰现有高风险异步链路：

- 不新增任何网络请求；
- 不新增 `watch` 依赖源；
- 不新增 `setInterval` / `setTimeout`；
- 不新增 DOM 事件监听；
- 不改变 ECharts 实例创建与销毁逻辑。

因此不会额外引入以下风险：

- 新旧请求结果覆盖导致的竞态错乱；
- 延时回调造成的状态覆盖；
- 事件监听重复注册与泄漏；
- loading 无法退出；
- 图表重复初始化造成的资源堆积。

唯一新增风险集中在“规则不合理导致建议误导性偏强”，通过以下方式控制：

- 横盘优先级高于弱趋势；
- 信号冲突时默认观望；
- 目标价和止损价必须满足方向正确性校验；
- 所有建议必须携带依据与风险提示。

## 验证方案

### 单元测试

在 `server/analysis.test.js` 中新增或扩展测试，覆盖：

- 多头共振时输出买入建议；
- 空头共振时输出卖出建议；
- 横盘信号 >= 3 时输出观望建议；
- 目标价方向不合理时自动降级为观望；
- 数据不足或冲突时输出低置信度观望。

### 前端自测

至少验证以下主路径和失败路径：

- 成功加载分析结果时正常展示建议卡；
- 买入 / 卖出 / 观望三种视觉状态正确；
- `null` 目标价和止损价时页面文案正常；
- 接口失败时不影响原有错误态；
- 切换标的和手动刷新时建议内容随最新分析刷新。

## 实施边界

- 仅修改 `server/analysis.js`、`server/analysis.test.js`、`src/market-types.ts`、`src/components/TrendDashboard.vue`
- 不新增依赖
- 不做无关重构

## 验收标准

- 看板中出现清晰的买入 / 卖出 / 观望建议卡；
- 建议同时展示目标价、止损 / 失效价、持续时间和依据；
- 所有建议都基于现有指标数据生成，不新增外部数据源；
- 横盘或冲突场景不会强行给出激进建议；
- 相关测试通过，且不影响现有刷新、错误态和图表渲染逻辑。
