# 股票搜索目录持久化设计

## 背景

当前左侧目录数据来自前端静态文件 `src/instruments.ts`，只能展示预置标的，无法在运行时新增或删除。用户希望在“切换浅色主题”按钮下方增加股票代码输入框，搜索后将股票加入目录，并支持删除，同时将相关数据集中保存在项目内的新数据目录下。

## 目标

- 在左侧主题切换按钮下方新增股票代码输入框与搜索按钮。
- 用户搜索成功后，将对应股票加入左侧目录并持久化到项目数据目录。
- 左侧目录中的股票支持删除，删除后同步更新持久化文件。
- 页面刷新或服务重启后，目录仍能恢复上次保存的数据。
- 保持现有行情与分析轮询链路可用，不引入旧响应覆盖新状态的问题。

## 非目标

- 不修改趋势分析计算逻辑和图表组件结构。
- 不引入数据库、状态管理库或新的 UI 组件库。
- 不支持模糊搜索、联想提示、批量导入或分组管理。

## 数据目录方案

新增目录 `data/watchlist/`，用于存放运行期目录数据。

建议文件结构如下：

- `data/watchlist/stocks.json`：左侧目录数据的唯一持久化文件。

文件内容采用数组结构，每项字段与前端目录渲染保持一致：

```json
[
  {
    "symbol": "159980",
    "displayName": "大成有色ETF",
    "market": "SZ",
    "description": "有色金属主题"
  }
]
```

## 方案

### 前端目录来源

左侧目录从“静态导入”切换为“页面启动后通过接口加载”。`src/instruments.ts` 保留为默认回退数据与初始展示数据，避免目录接口首次失败时页面完全空白。

### 搜索行为

用户输入 6 位股票代码后点击搜索或按回车，前端调用新增的目录写入接口。

服务端收到代码后：

1. 先校验股票代码格式；
2. 调用现有行情接口能力获取股票名称，作为新增目录项的展示名称；
3. 推断市场字段 `SH/SZ`；
4. 如果目录中不存在该股票，则写入 `stocks.json`；
5. 返回最新目录和本次命中的目录项。

前端在搜索成功后：

- 用接口返回的最新目录替换本地目录状态；
- 将当前激活标的切换为本次搜索股票；
- 清空输入框和错误提示；
- 保持现有 `AbortController + requestId` 机制，确保旧行情响应不会覆盖新股票状态。

### 删除行为

每个目录项新增删除按钮，点击删除时：

- 只触发删除，不触发目录切换；
- 调用新增删除接口；
- 删除成功后以服务端返回的最新目录为准更新前端状态。

若删除的是当前激活股票：

- 优先切换到删除项后的下一个股票；
- 若后方没有，则切到前一个股票；
- 若目录为空，则进入空目录状态，停止新的行情和分析请求启动，页面展示空态提示。

### 服务端存储边界

新增服务端目录存储模块，负责：

- 确保 `data/watchlist/` 与 `stocks.json` 存在；
- 读取目录数据；
- 追加目录项；
- 删除目录项；
- 对连续写操作做串行化，避免用户快速连续搜索/删除时发生文件覆盖。

写入逻辑采用单进程串行 promise 队列，保证同一时刻只存在一个落盘任务。

### 接口设计

- `GET /api/instruments`
  - 返回当前目录列表。
- `POST /api/instruments`
  - 请求体：`{ "symbol": "513100" }`
  - 返回：最新目录列表和本次命中的目录项。
- `DELETE /api/instruments?symbol=513100`
  - 返回：最新目录列表。

接口统一沿用现有 `ApiResponse<T>` 返回结构。

## 状态与异常处理

### 前端状态

新增以下状态：

- `instrumentsLoading`：目录初始化加载中；
- `instrumentsError`：目录加载失败；
- `searchKeyword`：搜索输入值；
- `searchLoading`：搜索提交中；
- `searchError`：搜索失败信息；
- `deleteLoadingSymbol`：正在删除的股票代码。

### 失败路径

- 目录初始化失败：显示错误提示，保留前端默认目录，允许用户重试。
- 搜索失败：在输入框附近显示错误原因，不改动当前目录。
- 删除失败：保留当前目录，并在侧栏显示失败提示。
- 空目录：右侧主内容显示“请先搜索并添加股票”的空态，不发起新的轮询请求。

## 风险与稳定性分析

本次改动同时涉及前端异步请求和服务端文件读写，需要重点控制以下风险：

- 快速连续搜索导致多次写文件并发执行，后写入覆盖前写入。
- 搜索成功切换激活股票后，旧的行情/分析请求回包覆盖新股票数据。
- 删除按钮嵌套在目录项点击区域中，若事件冒泡未拦截，可能出现“先删除又切换”的顺序错乱。
- 目录为空时仍保留轮询定时器回调，如果回调未判断空标的，会产生无效请求或错误状态残留。

对应控制措施：

- 服务端使用写入串行队列，所有增删改都进入同一链路执行。
- 前端继续沿用现有 `requestId` 与 `AbortController` 取消旧请求。
- 删除按钮显式阻止默认行为和事件冒泡。
- 所有抓取函数与定时器回调在无激活股票时直接返回，并主动回收 loading 状态。

## 验收标准

- 主题按钮下方出现股票代码输入框和搜索按钮。
- 输入合法股票代码并搜索成功后，目录新增该股票并自动切换到该股票。
- 刷新页面或重启服务后，已添加股票仍存在。
- 目录项支持删除，删除后页面状态保持正确。
- 删除当前激活股票时，页面自动切换到合理的下一个标的或进入空态。
- 连续搜索或删除不会出现重复目录项、旧数据覆盖或 loading 无法退出的问题。
- `npm run build` 与相关 Node 测试通过。
