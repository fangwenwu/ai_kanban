# 行情看板目录新增设计

## 背景

当前“行情看板”左侧目录由前端静态配置 `src/instruments.ts` 提供，页面基于该配置自动渲染目录项，并在切换时触发行情与分析请求。

本次需求是在现有目录中新增两只 ETF：

- 富国标普油气ETF `513350`
- 华夏电网设备ETF `159326`

## 目标

- 在不改变现有交互和异步刷新逻辑的前提下，为行情看板目录补充两只新标的。
- 保持现有条目顺序不变，并按需求顺序将新增项追加到目录中。
- 通过最小必要测试保证新增配置不会在后续改动中丢失。

## 非目标

- 不调整现有目录排序。
- 不修改已有标的名称、描述、市场字段。
- 不改造请求层、轮询机制、状态管理或服务端接口。

## 方案

### 数据配置

在 `src/instruments.ts` 中追加两条 `InstrumentItem` 配置，继续沿用当前字段结构：

- `symbol`
- `displayName`
- `market`
- `description`

其中市场按代码归属填写：

- `513350` 使用 `SH`
- `159326` 使用 `SZ`

描述文案采用与标的名称一致的直观表述，不扩展额外业务语义。

### 展示与交互影响

`src/App.vue` 已通过 `instruments` 数组渲染左侧目录，并使用 `activeSymbol` 驱动后续请求：

- 新增配置后会自动出现在目录中；
- 点击后仍走既有 `switchInstrument -> watch(activeSymbol) -> fetchQuote/fetchAnalysis` 链路；
- 不需要修改定时轮询、AbortController、手动刷新或错误态逻辑。

### 风险与稳定性分析

本次变更仅新增静态配置，不触碰以下高风险点：

- 异步请求竞态控制；
- 定时器注册与销毁；
- 事件监听绑定与释放；
- 加载态、错误态与刷新态切换。

因此不会引入新的执行顺序错乱、回调时序异常、监听泄漏或重复触发风险。唯一主要风险是新增配置字段填写错误，导致目录显示异常或请求 symbol 不正确。

### 验证方案

更新现有 `server/instruments-config.test.js`，在保留原有断言的基础上，新增对以下配置的校验：

- 出现 `513350`
- 出现 `159326`
- 维持旧有 `513100` 替换规则不回退

## 实施边界

- 仅修改 `src/instruments.ts` 与 `server/instruments-config.test.js`
- 不引入新依赖
- 不做无关重构

## 验收标准

- 左侧目录新增展示“富国标普油气ETF 513350”和“华夏电网设备ETF 159326”
- 两条新增项顺序与需求一致
- 现有页面逻辑不变
- 相关测试通过
